## Claude Code 固有のルール

- **共通ルールは `AGENTS.md` に従う。** 本ファイルは Claude 固有の補足のみ。
- Claude Code は、**デザイン・文章・UI/UX・アクセシビリティ**の助言や修正に
  使われることがある。
- 実装を担当する場合もあるが、**Codex やユーザーの変更を勝手に巻き戻さない**。
- **「レビューのみ」と依頼された場合は、ファイルを変更しない**（指摘・提案だけ行う）。
- 明示的に依頼されない限り、**commit・push・merge・rebase・ブランチ削除をしない**。
- **`main` への push は本番公開につながる可能性がある**ため、push 前に必ず
  ブランチ・`git status`・差分・ビルド結果を確認し、本番反映の可能性を伝える。
- **未コミットの変更がある場合は停止して報告する**
  （別の AI やユーザーが作業中の可能性があるため、勝手に触らない）。

## 作業の進め方（Claude Code）

1. `AGENTS.md` の「作業開始時のルール」に従い、ブランチと `git status` を確認する。
2. 未コミット変更があれば、作業を始めず報告する。
3. デザイン・文章の変更は `docs/SITE_POLICY.md` の方針を守る。
4. クリニック情報は `docs/CLINIC_FACTS.md` の確定情報を使い、推測で書かない。
5. 変更後は `npm run build` を実行し、必要なら `npm run dev` でプレビュー確認する。
6. 変更ファイルと内容を要約して報告し、commit / push はユーザーの指示を待つ。
